49个项目管理过程-ITTO
  1. 49个项目管理过程-ITTO
    1. 8.2.1 过程概述——项目整合管理（统筹兼顾）
      1. 制定项目章程
        1. 输入 立项管理文件、协议、事业环境因素、组织过程资产
        2. 输出 项目章程、假设日志
        3. 技术 专家判断、数据收集、人际关系与团队技能、会议
      2. 制定项目管理计划
        1. 输入 项目章程、其他知识领域规划过程的输出、事业环境因素、组织过程资产
        2. 输出 项目管理计划（3个基准、10个子计划、绩效测量基准、生命周期、开发方法）
        3. 技术 专家判断、数据收集、人际关系与团队技能、会议
      3. 指导管理项目工作
        1. 输入 项目管理计划、项目文件、批准的变更请求、事业环境因素、组织过程资产
        2. 输出 可交付成果、工作绩效数据、问题日志、变更请求、项目管理计划更新、项目文件更新、组织过程资产更新
        3. 技术 专家判断、项目管理信息系统、会议
      4. 管理项目知识
        1. 输入 项目管理计划、项目文件、可交付成果、事业环境因素、组织过程资产
        2. 输出 经验教训登记册、项目管理计划更新、组织过程资产更新
        3. 技术 专家判断、知识管理、信息管理、人际关系与团队技能
      5. 监控项目工作
        1. 输入 项目管理计划、项目文件、工作绩效信息、协议、事业环境因素、组织过程资产
        2. 输出 工作绩效报告、变更请求、项目管理计划更新、项目文件更新
        3. 技术 专家判断、数据分析、决策、会议
      6. 实施整体变更控制
        1. 输入 项目管理计划、项目文件、工作绩效报告、变更请求、事业环境因素、组织过程资产
        2. 输出 批准的变更请求、项目管理计划更新、项目文件更新
        3. 技术 专家判断、变更控制工具、数据分析、决策、会议
      7. 结束项目或阶段
        1. 输入 项目章程、项目管理计划、项目文件、验收的可交付成果、立项管理文件、协议、采购文档、组织过程资产
        2. 输出 最终产品、服务或成果、项目最终报告、项目文件更新、组织过程资产更新
        3. 技术 专家判断、数据分析、会议
    2. 9.2.1 过程概述——项目范围管理（做且只做）
      1. 规划范围管理
        1. 输入 项目章程、项目管理计划、事业环境因素、组织过程资产
        2. 输出 范围管理计划、需求管理计划
        3. 技术 专家判断、数据分析、会议
      2. 收集需求
        1. 输入 立项管理文件、项目章程、项目管理计划、项目文件、协议、事业环境因素、组织过程资产
        2. 输出 需求文件、需求跟踪矩阵
        3. 技术 专家判断、数据收集、数据分析、决策、数据表现、人际关系与团队技能、系统交互图、原型法
      3. 定义范围
        1. 输入 项目章程、项目管理计划、项目文件、事业环境因素、组织过程资产
        2. 输出 范围说明书、项目文件更新
        3. 技术 专家判断、数据分析、决策、人际关系与团队技能、产品分析
      4. 创建WBS
        1. 输入 项目管理计划、项目文件、事业环境因素、组织过程资产
        2. 输出 范围基准、项目文件更新
        3. 技术 分解、专家判断
      5. 确认范围
        1. 输入 项目管理计划、项目文件、工作绩效数据、核实的可交付成果
        2. 输出 验收的可交付成果、变更请求、工作绩效信息、项目文件更新
        3. 技术 检查、决策
      6. 控制范围
        1. 输入 项目管理计划、项目文件、工作绩效数据、组织过程资产
        2. 输出 工作绩效信息、变更请求、项目管理计划更新、项目文件更新
        3. 技术 数据分析
    3. 10.2.1 过程概述——项目进度管理（按时完成）
      1. 规划进度管理
        1. 输入 项目章程、项目管理计划、事业环境因素、组织过程资产
        2. 输出 进度管理计划
        3. 技术 专家判断、 数据技术、会议
      2. 定义活动
        1. 输入 进度管理计划、事业环境因素、组织过程资产
        2. 输出 活动清单、活动属性、里程碑清单、变更请求、项目管理计划更新
        3. 技术 专家判断、分解、滚动式规划、会议
      3. 排列活动顺序
        1. 输入 项目管理计划、项目文件、事业环境因素、组织过程资产
        2. 输出 项目进度网络图、项目文件更新
        3. 技术 紧前关系绘图法、箭线图法、确定和整合依赖关系、提前量与滞后量、项目管理信息系统
      4. 估算活动持续时间
        1. 输入 项目管理计划、项目文件、事业环境因素、组织过程资产
        2. 输出 持续时间估算、估算依据、项目文件更新
        3. 技术 专家判断、类比估算、参数估算、三点估算、自下而上估算、数据分析、决策、会议
      5. 制定进度计划
        1. 输入 项目管理计划、项目文件、协议、事业环境因素、组织过程资产
        2. 输出 进度基准、项目进度计划、进度数据、项目日历、变更请求、项目管理计划更新、项目文件更新
        3. 技术 进度网络分析、关键路径法、资源优化、数据分析、提前量与滞后量、进度压缩、计划评审技术、项目管理信息系统、敏捷或适应型发布规划
      6. 控制进度
        1. 输入 项目管理计划、项目文件、工作绩效数据、组织过程资产
        2. 输出 工作绩效信息、进度预测、变更请求、项目管理计更新、项目文件更新
        3. 技术 数据分析、关键路径法、项目管理信息系统、资源优化、提前量与滞后量、进度压缩
    4. 11.2.1 过程概述——项目成本管理（预算内完成）

      3. 制定预算
        1. 输入 项目管理计划、可行性研究文件、项目文件、协议、事业环境因素、组织过程资产

      4. 控制成本
        1. 输入 项目管理计划、项目资金需求、项目文件、工作绩效数据、组织过程资产

        3. 技术 专家判断、数据分析、完工尚需绩效指数、项目管理信息系统
    5. 12.2.1 过程概述——项目质量管理（满足要求）

        1. 输入 项目管理计划、项目文件、组织过程资产
